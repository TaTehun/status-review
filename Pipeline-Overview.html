Pipeline Steps

01
Login
Selenium launches Chrome, navigates to the TMS portal, and authenticates using credentials from a .env file.
02
Search
Filters by carrier ID, tender status (multi-select via JS), and date range set to TODAY.
03
Customize
Column layout configured via the site's own JS dual-listbox functions — unwanted columns removed, desired ones ordered.
04
Export
Export popup handled with window switching. File polled from disk for up to 60s, then renamed and saved.
05
Send
File formatted with openpyxl, HTML summary table built, sent via SMTP with the xlsx attached.

Key Technical Highlights

Hidden Select via JavaScript
The tender status dropdown is a hidden <select> — Selenium can't click it directly. Options are set via execute_script and the site's own showSelected() callback is called to sync the UI.
iframe Navigation
The TMS portal uses nested iframes (i2ui_shell_content → navFrame / resultFrame). Every interaction re-switches to the correct frame context to avoid stale element errors.
Popup Window Handling
The export submit button lives in a new browser window. The driver switches to the popup via window_handles, clicks submit, then closes the popup and returns to the main window.
Download Path via Chrome Prefs
Download directory is injected at launch via add_experimental_option("prefs", {"download.default_directory": path}) — the only reliable method for forcing Chrome's save location when using chromedriver.
Column Customization via JS
The List Customization panel uses the site's own tmuiduallistboxmoveit function to move and reorder columns — replicated in JavaScript injected by the driver for full control over the final column layout.
File Rotation
Exports are saved as TenderStatus_MMDDYYYY.xlsx. On each run, the oldest files are pruned so the data folder never exceeds the configured limit. Same-day files are overwritten automatically.

Error Handling

Login Failure
Any exception during __init__ propagates to the top-level try/except. The browser is always closed in finally — no dangling Chrome processes.
Export Timeout
File polling runs for up to 60 seconds per attempt. If no file is found, the attempt returns None and the pipeline retries up to 5 times before giving up.
SMTP Failure
Missing credentials or server errors are caught, logged with full traceback, and return False. The pipeline does not crash — email failure is non-fatal.
Stale Frame Context
Every method calls switch_to_result() or switch_to_nav() before interacting with elements — prevents stale context errors after page transitions.
Same-Day Overwrite
If TenderStatus_MMDDYYYY.xlsx already exists, it is deleted before renaming the new export — prevents duplicate or locked-file errors on re-runs.
No Export File
If all 5 export attempts fail, exported_file remains None and the email block is skipped entirely. A warning is logged for visibility.
