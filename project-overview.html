// technical_highlights

AUTOMATION
Hidden Selects via JavaScript
Status and logistics group dropdowns are hidden <select> elements. Values set via execute_script + site's own showSelected() to sync the UI — Selenium click alone cannot reach them.
NAVIGATION
Nested iframe Re-entry
Portal wraps content in nested iframes. Frame context re-established before every interaction via switch_to_result() / switch_to_nav() to prevent stale element errors.
EXPORT
Popup Window Handling
Export submit opens a new browser window. Driver switches via window_handles, submits, polls disk for the file, closes popup, returns to main — all without user interaction.
DISPATCH
Multi-Region via sys.argv
One main.py, three bat files. Each passes a run key; the RUNS dict maps it to logistics groups, region label, and output filename.
COLUMN
Dual-Listbox API Injection
Column customization calls the site's own tmuiduallistboxmoveit function via execute_script — columns moved, filtered, and reordered to match the required layout.
EMAIL
HTML Summary in Body
Only Load ID and Status columns extracted and rendered as inline HTML table. Recipients see the summary without opening the attachment.

// error_handling

Login Failure
Exception propagates to top-level try/except. Browser always closed in finally — no orphaned Chrome processes.
Export Timeout
Polls disk for 60s per attempt. Returns None on timeout; retries up to 5×.
All Retries Failed
Failure alert sent to a separate recipient list with distinct subject — report recipients are not spammed.
SMTP Failure
Caught, logged, returns False. Pipeline does not crash.
Stale Frame Context
Frame re-established before every interaction. switch_to_result() / switch_to_nav() called explicitly at each step.
Same-Day Re-run
Existing file for same region + date deleted before rename. No locked-file errors on re-runs.

// dev_process · Claude Code Conversations

Hidden Select — Root Cause First
TJ
The status dropdown doesn't respond to Selenium's Select class. Inspected the element — it's hidden. What's the right approach?
C
The element is hidden so Selenium can't interact with it directly. The site exposes a JS callback — status_tmcombobox.showSelected() — that syncs the hidden select to the visible UI. Set values via execute_script, then call that function to commit the selection.
Multi-Region — One Entry Point
TJ
3 regions, each needs different logistics groups and its own output filename. Don't want 3 separate scripts — one entry point, one RUNS dict, bat file passes the key.
C
Done. RUNS maps each key to logistics groups, label, and region slug. sys.argv[1] resolves the config at runtime. Bat files are the only difference between region runs.
Popup Export — window_handles
TJ
Export opens a popup window with a Submit button. find_element throws — the button isn't in the main frame.
C
The popup is a separate browser window, not an iframe. Switch to it via window_handles[-1], click Submit, poll disk until the file appears, close the popup, then switch back to handles[0].
Failure Alert — Separate Recipients
TJ
If all 5 export attempts fail, I want an alert — but to a different list than the normal report recipients. Subject should be distinct too.
C
Added EMAIL_TO_FAILURE / EMAIL_SUBJECT_FAILURE in config. Success and failure paths are fully decoupled — neither list receives the other's messages.
